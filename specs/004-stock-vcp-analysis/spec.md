# Feature Specification: 单股票VCP分析查询

**Feature Branch**: `004-stock-vcp-analysis`  
**Created**: 2026-03-11  
**Status**: Draft  
**Input**: 用户描述: "我想实现输入单个股票，输出这个股票当前的 vcp 相关的分析。用中文！！！"

## Clarifications

### Session 2026-03-11

- Q: 查询接口形式（Web界面如何呈现以及是否需要其他访问方式）? → A: 集成到K线图页面，同时提供命令行脚本支持
- Q: 当VCP扫描结果已过期（超过7天）时，查询应该如何处理? → A: 显示缓存的过期数据并附带明显警告，用户可以查看但知道数据不是最新的
- Q: 命令行脚本执行VCP分析后，应该以什么格式输出结果? → A: 格式化的中文文本报告（类似现有analyze-stock-vcp.ts脚本的输出，包含表格、符号、颜色标记）
- Q: VCP分析功能在K线图页面的具体集成方式是什么? → A: K线图页面提供生成VCP分析报告的按钮和查看生成状态，生成后的完整分析文档在新的独立页面中查看

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 快速生成单个股票的VCP分析 (Priority: P1)

投资者想要快速了解某只股票（如"德业股份"或"600519"）当前的VCP形态状态。用户在K线图页面点击"生成VCP分析报告"按钮，系统在后台生成分析报告并显示生成状态，完成后在新页面展示该股票的VCP分析结果，包括是否处于VCP形态、收缩阶段、回调状态等关键信息。

**Why this priority**: 这是核心功能，为用户提供即时的VCP形态判断，是投资决策的基础信息。

**Independent Test**: 用户在K线图页面点击生成按钮，系统在3秒内完成分析并在新页面展示VCP分析报告，包括形态有效性判断和关键指标。

**Acceptance Scenarios**:

1. **Given** 用户在K线图页面，**When** 点击"生成VCP分析报告"按钮，**Then** 系统显示"生成中..."状态，并在后台执行VCP分析
2. **Given** VCP分析生成完成，**When** 系统完成计算，**Then** 自动打开新页面展示完整的分析报告，K线图页面显示"已完成"状态
3. **Given** 系统显示VCP分析结果，**When** 用户在新页面查看报告，**Then** 所有指标说明均使用中文，数值格式符合中国用户习惯（如百分比保留2位小数）

---

### User Story 2 - 查看收缩阶段详细信息 (Priority: P2)

用户需要了解股票的VCP收缩阶段细节，以判断形态的质量。系统展示每个收缩阶段的高点、低点、幅度、持续时间和平均成交量，帮助用户评估收缩是否逐步缩小、成交量是否萎缩。

**Why this priority**: 收缩阶段是VCP形态的核心特征，详细信息帮助用户判断形态质量和入场时机。

**Independent Test**: 查询一只有多个收缩阶段的股票，系统按时间顺序列出所有收缩阶段，每个阶段显示完整的统计数据（日期范围、价格范围、幅度、持续天数、平均成交量）。

**Acceptance Scenarios**:

1. **Given** 股票有3个收缩阶段，**When** 用户查看分析结果，**Then** 系统按时间顺序显示3个收缩阶段，每个显示：开始日期、结束日期、高点价格、低点价格、收缩幅度(%)、持续天数、平均成交量
2. **Given** 用户查看收缩阶段列表，**When** 收缩幅度逐步减小（如35% → 20% → 12%），**Then** 系统高亮标注或提示"收缩幅度递减，符合VCP特征"
3. **Given** 收缩阶段成交量数据可用，**When** 用户查看，**Then** 系统显示每个收缩阶段的平均成交量，并与前期对比标注萎缩程度

---

### User Story 3 - 查看回调阶段分析 (Priority: P2)

用户需要识别股票在上涨趋势中的回调，以寻找低吸机会。系统展示回调阶段的高点、低点、回调幅度、持续时间，并标注当前是否处于回调中，距离回调低点有多少天。

**Why this priority**: 回调是上涨趋势中的买入机会，及时识别回调状态对交易决策至关重要。

**Independent Test**: 查询一只正在回调的股票，系统显示当前回调状态（如"正在回调中"或"3天前到达最低点"），并展示回调的完整数据。

**Acceptance Scenarios**:

1. **Given** 股票当前正在回调，**When** 用户查询，**Then** 系统显示"🔴 正在回调中"状态，并展示本次回调的高点、当前低点、回调幅度、已持续天数
2. **Given** 股票3天前完成回调，**When** 用户查询，**Then** 系统显示"🟡 3天前到达最低点，可能开始反弹"，并展示从最低点的反弹幅度和距回调高点的距离
3. **Given** 股票有多次历史回调，**When** 用户查看回调列表，**Then** 系统按时间顺序显示所有回调，并标注每次回调是否发生在上升趋势中

---

### User Story 4 - 查看趋势模板和RS评分 (Priority: P3)

用户需要了解股票是否符合Mark Minervini的趋势模板标准，以及相对强度评分（RS Rating）。系统显示趋势模板检查结果（通过/未通过）和RS评分（0-100），帮助用户评估股票的趋势强度。

**Why this priority**: 趋势模板和RS评分是辅助指标，用于进一步验证股票质量，但不是VCP形态判断的核心。

**Independent Test**: 查询任意股票，系统显示趋势模板状态（✅通过 或 ❌未通过）和RS评分数值（如85），以及与52周高低点的距离。

**Acceptance Scenarios**:

1. **Given** 股票符合趋势模板，**When** 用户查询，**Then** 系统显示"✅ 趋势模板：通过"，并列出关键指标（如股价高于200日均线、50日均线上穿200日均线等）
2. **Given** 股票RS评分为85，**When** 用户查看，**Then** 系统显示"RS评分：85/100"，并注释"相对市场表现优秀"
3. **Given** 股票距52周高点-5%，**When** 用户查询，**Then** 系统显示"距52周高点：-5.00%"，并标注"接近历史高点"

---

### User Story 5 - 查看最近K线数据 (Priority: P3)

用户需要查看股票最近10-20天的K线数据，包括开盘价、最高价、最低价、收盘价、涨跌幅和成交量，以便结合VCP分析判断短期趋势。

**Why this priority**: K线数据提供短期价格走势的直观信息，是对VCP分析的有益补充，但不是核心需求。

**Independent Test**: 查询任意股票，系统在分析结果下方显示最近10天的K线数据表格，包含完整的OHLC和成交量数据。

**Acceptance Scenarios**:

1. **Given** 用户查询股票，**When** 滚动到K线数据区域，**Then** 系统显示最近10天的K线数据表格，包含日期、开盘、最高、最低、收盘、涨跌幅、成交量
2. **Given** K线数据中某天涨跌幅为正，**When** 用户查看，**Then** 该天涨跌幅用绿色显示（中国股市习惯）；跌幅用红色显示
3. **Given** 用户需要更多历史数据，**When** 点击"查看更多K线"，**Then** 系统展开显示最近30天或60天的数据

---

### Edge Cases

- **股票代码不存在**: 用户输入无效的股票代码（如"999999"），系统提示"未找到股票代码 999999，请检查输入"
- **股票无K线数据**: 查询的股票在数据库中存在但无K线数据（如新上市股票），系统提示"该股票暂无足够的K线数据进行VCP分析（需要至少30天数据）"
- **VCP扫描结果过期**: 股票的最新VCP扫描结果超过7天未更新，系统显示警告"⚠️ 扫描结果已过期（最后扫描日期：2026-03-04），请等待下次自动更新或手动触发扫描"，但仍然展示缓存的VCP分析数据（包括收缩阶段、回调阶段等信息）
- **无VCP形态**: 股票当前不符合VCP形态标准，系统显示"❌ 当前无有效VCP形态"，并列出原因（如"收缩次数不足"或"收缩幅度未递减"）
- **输入股票名称模糊匹配**: 用户输入"德业"，系统找到多只匹配股票（如"德业股份"、"德邦股份"），显示候选列表让用户选择
- **查询速度慢**: 股票K线数据超过1000条，分析计算耗时超过3秒，系统显示加载动画和进度提示"正在分析中...（已完成60%）"
- **重复生成**: 用户在上一次VCP分析报告生成过程中再次点击生成按钮，系统提示"正在生成中，请稍候"并阻止重复提交
- **网络错误**: 查询时后端服务不可用，系统显示友好错误提示"服务暂时不可用，请稍后重试"，并提供重试按钮

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须支持通过股票代码（如"605117"或"605117.SH"）查询VCP分析结果
- **FR-002**: 系统必须支持通过股票名称（如"德业股份"）模糊匹配并查询VCP分析结果
- **FR-003**: 系统必须在查询时调用后端VCP分析服务，获取实时分析结果（基于最新可用的K线数据）
- **FR-004**: 系统必须显示VCP形态有效性判断（有效/无效），并列出判断依据（收缩次数、幅度递减、成交量萎缩等）
- **FR-005**: 系统必须显示所有收缩阶段的详细信息，包括：开始和结束日期、高点和低点价格、收缩幅度(%)、持续天数、平均成交量
- **FR-006**: 系统必须显示所有回调阶段的详细信息，包括：开始和结束日期、高点和低点价格、回调幅度(%)、持续天数、是否在上升趋势中、当前距回调低点的天数
- **FR-007**: 系统必须标注当前股票的回调状态："正在回调中"、"N天前到达最低点"或"回调结束"
- **FR-008**: 系统必须显示趋势模板检查结果（通过/未通过）和关键指标（如MA200、MA50关系）
- **FR-009**: 系统必须显示RS评分（0-100）和与52周高低点的距离百分比
- **FR-010**: 系统必须显示最近10天的K线数据表格，包含：日期、开盘价、最高价、最低价、收盘价、涨跌幅(%)、成交量（手）
- **FR-011**: 系统必须在查询无效股票代码时，返回明确的错误提示信息
- **FR-012**: 系统必须在股票K线数据不足30天时，提示用户无法进行VCP分析
- **FR-013**: 所有界面文本、指标名称、提示信息必须使用中文
- **FR-014**: 所有数值格式必须符合中国用户习惯（金额用逗号分隔千位、百分比保留2位小数、成交量以"手"为单位）
- **FR-015**: 系统必须在查询结果中标注扫描日期，当扫描结果超过7天时显示明显的过期警告，但仍然展示缓存的分析数据供用户参考
- **FR-016**: K线图页面必须提供"生成VCP分析报告"按钮，用户点击后触发VCP分析生成
- **FR-017**: K线图页面必须显示VCP分析报告的生成状态（如"生成中..."、"已完成"、"生成失败"）
- **FR-018**: VCP分析报告生成完成后，必须在新的独立页面中展示完整的分析结果（包括所有收缩阶段、回调阶段、K线数据等）
- **FR-019**: 系统必须提供命令行脚本支持，允许用户通过终端命令（如`npm run analyze-vcp <股票代码>`）执行VCP分析，并以格式化的中文文本报告输出结果（包含表格、状态符号、分段显示收缩阶段、回调阶段、K线数据等）

### Key Entities

- **股票 (Stock)**: 代表A股市场中的上市公司股票，包含股票代码、股票名称、上市日期等基本信息
- **VCP扫描结果 (VcpScanResult)**: 存储股票的VCP分析结果快照，包括扫描日期、趋势模板状态、收缩次数、RS评分、回调状态、52周高低点距离等
- **K线数据 (KLineData)**: 股票的历史价格和成交量数据，包含日期、开盘价、最高价、最低价、收盘价、成交量
- **收缩阶段 (Contraction)**: VCP形态中的收缩区间，记录高点和低点的日期和价格、收缩幅度、持续天数、平均成交量
- **回调阶段 (Pullback)**: 上涨趋势中的价格回调区间，记录高点和低点的日期和价格、回调幅度、持续天数、是否在上升趋势中

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户输入有效股票代码后，系统在3秒内返回VCP分析结果（包含所有必要信息）
- **SC-002**: 对于有VCP形态的股票，分析结果准确率达到95%以上（与人工判断对比）
- **SC-003**: 用户能够在不阅读任何说明文档的情况下，完成股票查询并理解分析结果（所有文本使用中文，指标有清晰注释）
- **SC-004**: 系统支持至少1000只A股的VCP分析查询，无性能衰减
- **SC-005**: 当股票无VCP形态或数据不足时，用户能够清楚了解原因（错误提示准确率100%）
- **SC-006**: 90%的用户能够通过查看收缩阶段和回调阶段信息，独立判断VCP形态质量和交易时机
- **SC-007**: 查询结果的数据新鲜度在7天以内（95%的查询返回7天内扫描的结果）
- **SC-008**: 系统在查询高峰期（如开盘前）支持至少50个并发查询，响应时间不超过5秒
