## 第四步：检查收缩深度是否递减

### 原始深度序列：
```
8.01% → 13.87% → 9.44% → 10.86% → 6.87%
```

**问题**: 第2次（13.87%）比第1次（8.01%）大了！❌

### 但是！过滤掉 < 5% 的收缩后：

由于所有收缩都 ≥ 5%，都保留了。但VCP算法会检查**整体趋势**。

让我们看另一个角度 - **从第2次开始看**：

```
13.87% → 9.44% → 10.86% → 6.87%
```

还是不完美递减...

### 实际判断逻辑：

代码中的判断可能不是严格递减，而是看**最后几次收缩是否收窄**：

```
最近3次收缩：
9.44% → 10.86% → 6.87%
              └─→ 最后一次最小 ✓
```

或者算法中允许一定的波动，只要**整体趋势向下**就行。
