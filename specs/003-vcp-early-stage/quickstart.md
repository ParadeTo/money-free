# 快速开始：VCP早期启动阶段股票筛选

**功能**: 003-vcp-early-stage  
**日期**: 2026-03-11  
**目标用户**: 开发者、测试人员

## 概述

本功能提供两种使用方式：
1. **Web界面**：在VCP选股页面通过可视化筛选器操作
2. **命令行工具**：通过脚本快速筛选并导出报告

## 前置条件

### 系统要求
- Node.js 20.x（后端）
- Node.js 18+（前端）
- nvm（用于命令行工具）
- SQLite 3.40+（已有数据）

### 数据要求
- 数据库中必须已有VCP扫描结果（`vcpScanResult`表）
- 至少有252天的K线数据用于计算52周高低点

## Web界面使用

### 1. 启动应用

```bash
# 启动后端（Terminal 1）
cd backend
nvm use 20
npm run start:dev

# 启动前端（Terminal 2）
cd frontend
npm run dev
```

### 2. 访问页面

打开浏览器访问：`http://localhost:5173`（或配置的前端端口）

导航到：**VCP选股** 页面

### 3. 使用筛选器

1. 在页面顶部找到"**早期启动筛选**"区域
2. 调整筛选条件：
   - **距52周低点**：拖动滑块（20%-60%），默认40%
   - **距52周高点**：拖动滑块（10%-50%），默认30%
   - **收缩次数范围**：拖动双端滑块（2-8次），默认3-4次
3. 点击"**筛选**"按钮
4. 查看结果：
   - 绿色高亮：收缩中的股票（最早期）
   - 普通显示：回调中/回调结束的股票
5. 如果显示提示（结果<5或>30只）：
   - 点击"放宽5%"或"收紧5%"快捷按钮
   - 或手动调整滑块

### 4. 导出结果

点击"**导出**"按钮，选择格式：
- Excel (.xlsx)
- CSV (.csv)

## 命令行工具使用

### 1. 基本用法（使用默认条件）

```bash
cd backend
./show-vcp-early.sh
```

输出：终端表格，按距52周低点排序

### 2. 自定义筛选条件

```bash
cd backend
./show-vcp-early.sh \
  --low-threshold 35 \
  --high-threshold 25 \
  --min-contraction 2 \
  --max-contraction 5
```

### 3. 导出到Markdown文档

```bash
cd backend
./export-vcp-early.sh \
  --low-threshold 40 \
  --high-threshold 30
```

输出文件：`docs/vcp/daily-reports/VCP选股-早期启动-2026-03-10.md`

### 4. 查看帮助

```bash
cd backend
./show-vcp-early.sh --help
```

## 筛选条件说明

### 距52周低点（distFrom52WeekLow）

- **定义**：当前价格距离过去52周最低价的百分比
- **计算**：`(当前价 - 52周低) / 52周低 × 100`
- **示例**：52周低10元，当前价14元 → 40%
- **推荐值**：
  - **40%**（默认）：适中，平衡早期性和安全性
  - **20-30%**：更早期，风险较高
  - **50-60%**：更安全，但可能错过最早期机会

### 距52周高点（distFrom52WeekHigh）

- **定义**：当前价格距离过去52周最高价的百分比
- **计算**：`(52周高 - 当前价) / 52周高 × 100`
- **示例**：52周高20元，当前价14元 → 30%
- **推荐值**：
  - **30%**（默认）：有较大上涨空间
  - **40-50%**：上涨空间更大，但可能处于深度调整
  - **10-20%**：接近高点，适合强势股

### 收缩次数范围（contractionCount）

- **定义**：VCP形态中价格收缩的次数
- **含义**：
  - **2-3次**：非常早期，形态刚开始
  - **3-4次**（默认）：典型早期，形态初步成型
  - **4-5次**：中期，形态较成熟
  - **6-8次**：后期，形态完全成熟
- **推荐值**：**3-4次**，平衡早期性和形态可靠性

## 结果解读

### VCP阶段含义

1. **📊 收缩中**（最优）
   - 股票还在价格收缩整理阶段
   - 还未进入回调，最早期
   - 风险：需要等待回调后的买点

2. **⚡ 回调中**（次优）
   - 股票正在上升趋势中回调
   - 今天就是回调低点或接近低点
   - 机会：如果明天企稳，可能是买点

3. **✅ 回调结束**（观察）
   - 回调已结束，正在反弹
   - 已经开始上涨
   - 适合：确认趋势后追涨

### 表格排序

结果按以下规则排序：
1. **一级排序**：VCP阶段（收缩中 > 回调中 > 回调结束）
2. **二级排序**：距52周低点（从近到远）

**示例**：
```
排名  股票    阶段      距52周低%
1     A股     收缩中    25%       ← 最早期+最接近低点
2     B股     收缩中    35%       
3     C股     回调中    30%       ← 阶段次优，但接近低点
4     D股     回调结束  28%
```

## 常见场景

### 场景1：寻找极早期股票

**目标**：找到刚开始形成VCP的股票

**设置**：
- 距52周低点：25-35%（更接近底部）
- 距52周高点：40-50%（更大空间）
- 收缩次数：2-3次（更早期）

**预期**：筛选结果可能很少（<5只），但都是极早期机会

### 场景2：寻找稳健早期股票

**目标**：平衡早期性和形态可靠性

**设置**：
- 距52周低点：35-45%（默认附近）
- 距52周高点：25-35%
- 收缩次数：3-4次（默认）

**预期**：筛选结果5-15只，形态较成熟

### 场景3：市场整体高位时的策略

**目标**：市场普遍涨幅较大时，仍想找机会

**设置**：
- 距52周低点：50-60%（放宽到最大）
- 距52周高点：20-30%（接受接近高点）
- 收缩次数：3-5次

**预期**：筛选结果可能较多（20-30只），需要结合基本面二次筛选

## 故障排除

### 问题1：Web界面没有筛选器

**原因**：功能尚未部署或前端未正确加载

**解决**：
1. 检查前端console是否有错误
2. 确认后端API是否运行：`curl http://localhost:3000/api/vcp/early-stage`
3. 清除浏览器缓存并刷新

### 问题2：筛选结果为空

**原因**：筛选条件过严或数据库无VCP扫描结果

**解决**：
1. 放宽筛选条件（距52周低点提高到50-60%）
2. 检查数据库：`cd backend && npx prisma studio`
3. 运行VCP扫描：`cd backend && npm run calculate-vcp`

### 问题3：命令行脚本报错

**原因**：Node版本不是20.x或依赖未安装

**解决**：
```bash
nvm use 20
cd backend
npm install
./show-vcp-early.sh
```

### 问题4：localStorage不生效

**原因**：浏览器隐私模式或禁用localStorage

**解决**：
1. 退出隐私模式
2. 检查浏览器设置：允许网站存储数据
3. 打开DevTools → Application → Local Storage，检查是否有 `money-free:vcp-early-filter:v1`
