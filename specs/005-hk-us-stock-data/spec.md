# Feature Specification: 港股和美股核心股票数据支持

**Feature Branch**: `005-hk-us-stock-data`  
**Created**: 2026-03-12  
**Status**: Draft  
**Input**: User description: "增加港股和美股核心股票的数据"

## Clarifications

### Session 2026-03-12

- Q: How should the system handle changes in index composition over time? → A: Periodic re-import with manual triggering - Admin manually triggers re-fetch when needed, system adds new index members and keeps historical data for removed stocks (no automatic scheduled updates)
- Q: Where should trading calendar data come from for each market? → A: Market data inference - System infers trading days from actual data returned by APIs, no separate trading calendar maintenance needed
- Q: If the import process fails partway through, how should the system handle recovery? → A: Resume from checkpoint - Track import progress and skip already-imported stocks, continue from where it stopped
- Q: How should stock names be displayed in the UI for HK and US stocks? → A: Original language only - Display stock names as provided by data source without translation (HK stocks in Chinese, US stocks in English)
- Q: When should the system switch from AkShare to Yahoo Finance for a specific stock? → A: Smart hybrid - Try AkShare first for each stock, automatically fall back to Yahoo Finance on failure, and record which source successfully provided data
- Q: What should happen when both data sources fail for a specific stock? → A: Skip and log - Skip the failed stock, log detailed error information, continue processing remaining stocks, generate comprehensive failure report at completion

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 导入港股核心股票数据 (Priority: P1)

投资者需要分析港股市场的优质股票（如腾讯、阿里巴巴、美团等），但系统目前只支持A股数据。用户运行港股数据导入工具，系统从数据源获取恒生指数成分股或恒生科技指数成分股的基本信息和历史K线数据，并保存到数据库中。导入完成后，用户可以在股票列表中看到这些港股。

**Why this priority**: 港股包含大量优质中概股，是中国投资者重要的投资市场，支持港股数据是系统国际化的第一步。

**Independent Test**: 运行港股数据导入脚本，系统成功导入约110只恒生指数和恒生科技指数成分股的基本信息和最近10年的日K线数据，用户可以通过股票代码（如"00700.HK"）查询到腾讯控股的股票信息和完整历史数据。

**Acceptance Scenarios**:

1. **Given** 系统只有A股数据，**When** 管理员运行港股数据导入脚本，**Then** 系统从数据源获取核心港股列表（恒生指数和恒生科技指数成分股，去重后约110只），并显示导入进度和成功数量
2. **Given** 港股基本信息导入成功，**When** 系统继续导入K线数据，**Then** 每只股票导入最近10年的日K线数据（包括开盘价、最高价、最低价、收盘价、成交量），并以港币为单位存储
3. **Given** 港股数据导入完成，**When** 用户在股票列表页面搜索"腾讯"或"00700"，**Then** 系统返回"腾讯控股 (00700.HK)"，并显示其市场标识为"HK"

---

### User Story 2 - 导入美股核心股票数据 (Priority: P1)

投资者需要分析美股市场的核心股票（如苹果、微软、特斯拉、英伟达等），系统需要支持美股数据。用户运行美股数据导入工具，系统从数据源获取标普500或纳斯达克100成分股的基本信息和历史K线数据，并保存到数据库中。导入完成后，用户可以在股票列表中看到这些美股。

**Why this priority**: 美股是全球最大的股票市场，包含科技巨头和行业领军企业，支持美股数据对投资者分析全球市场至关重要。

**Independent Test**: 运行美股数据导入脚本，系统成功导入约550只标普500和纳斯达克100成分股的基本信息和最近10年的日K线数据，用户可以通过股票代码（如"AAPL"）查询到苹果公司的股票信息和完整历史数据。

**Acceptance Scenarios**:

1. **Given** 系统只有A股和港股数据，**When** 管理员运行美股数据导入脚本，**Then** 系统从数据源获取核心美股列表（标普500和纳斯达克100成分股，去重后约550只），并显示导入进度和成功数量
2. **Given** 美股基本信息导入成功，**When** 系统继续导入K线数据，**Then** 每只股票导入最近10年的日K线数据（包括开盘价、最高价、最低价、收盘价、成交量），并以美元为单位存储
3. **Given** 美股数据导入完成，**When** 用户在股票列表页面搜索"Apple"或"AAPL"，**Then** 系统返回"Apple Inc. (AAPL)"，并显示其市场标识为"US"

### User Story 3 - 查询和分析港股、美股VCP形态 (Priority: P2)

投资者想要对港股或美股执行VCP形态分析，就像分析A股一样。用户在K线图页面选择一只港股（如"00700.HK"）或美股（如"AAPL"），点击"生成VCP分析报告"按钮，系统对该股票执行VCP分析，并在新页面展示收缩阶段、回调阶段、趋势模板等信息。

**Why this priority**: VCP分析是系统的核心功能，支持港股和美股后，用户应该能够对这些市场的股票执行相同的分析功能。

**Independent Test**: 用户选择港股"腾讯控股 (00700.HK)"，点击生成VCP分析报告，系统在5秒内完成分析并在新页面展示该股票的VCP形态、收缩阶段、回调状态等信息。

**Acceptance Scenarios**:

1. **Given** 用户在K线图页面查看港股"00700.HK"，**When** 点击"生成VCP分析报告"按钮，**Then** 系统基于港股的K线数据执行VCP形态识别，并展示分析结果
2. **Given** 用户在K线图页面查看美股"AAPL"，**When** 点击"生成VCP分析报告"按钮，**Then** 系统基于美股的K线数据执行VCP形态识别，并展示分析结果
3. **Given** 港股或美股VCP分析完成，**When** 用户查看分析结果，**Then** 所有价格数据正确显示货币单位（港股显示"HKD"或"港币"，美股显示"USD"或"美元"），并保留2位小数

### User Story 4 - 查看跨市场股票列表 (Priority: P2)

投资者想要在股票列表中同时查看A股、港股和美股，并能够根据市场进行筛选。用户在股票列表页面可以看到所有市场的股票，并通过市场筛选器（"A股"、"港股"、"美股"）快速过滤感兴趣的市场。

**Why this priority**: 多市场支持后，用户需要便捷的方式在不同市场之间切换和浏览。

**Independent Test**: 用户打开股票列表页面，看到混合的A股、港股、美股列表（每只股票旁边显示市场标识），点击"港股"筛选器后，列表只显示港股。

**Acceptance Scenarios**:

1. **Given** 系统已导入A股、港股、美股数据，**When** 用户打开股票列表页面，**Then** 系统默认显示所有市场的股票，每只股票旁边显示市场标识（"SH"、"SZ"、"HK"、"US"）
2. **Given** 用户在股票列表页面，**When** 点击市场筛选器选择"港股"，**Then** 列表只显示market字段为"HK"的股票
3. **Given** 用户在股票列表页面，**When** 搜索框输入"腾讯"，**Then** 系统返回匹配的港股"腾讯控股 (00700.HK)"和可能的A股相关股票

---

### User Story 5 - 增量更新港股和美股K线数据 (Priority: P3)

系统需要定期更新港股和美股的K线数据，以保持数据时效性。管理员运行增量更新脚本，系统自动检查每只港股和美股的最新K线数据日期，并从数据源拉取最新的K线数据（直到当前交易日）。

**Why this priority**: 数据的时效性是分析准确性的基础，但相比初次导入，增量更新的优先级较低。

**Independent Test**: 手动修改某只港股的最新K线数据日期为7天前，运行增量更新脚本，系统检测到该股票数据过期，自动拉取最近7天的K线数据并更新数据库。

**Acceptance Scenarios**:

1. **Given** 港股"00700.HK"的最新K线数据日期为2026-03-05，**When** 运行增量更新脚本（当前日期2026-03-12），**Then** 系统拉取2026-03-06至2026-03-12的K线数据并保存到数据库
2. **Given** 美股"AAPL"的最新K线数据已是最新交易日，**When** 运行增量更新脚本，**Then** 系统跳过该股票，显示"AAPL 数据已是最新"
3. **Given** 系统有100只港股和200只美股，**When** 运行全量增量更新，**Then** 系统批量处理所有股票（每批10只），显示进度条和预计剩余时间

### Edge Cases

- **股票代码格式差异**: 港股代码格式为"00700.HK"（5位数字+.HK后缀），美股代码格式为"AAPL"（纯字母），系统需要正确识别和存储不同格式的股票代码
- **数据源不可用**: 导入过程中，AkShare返回错误（如超时、限流），系统自动切换到Yahoo Finance尝试获取该股票数据；如果两个数据源都失败，系统记录失败的股票代码和原因
- **数据源混合使用**: 导入100只美股时，60只通过AkShare成功导入，40只AkShare失败后通过Yahoo Finance成功导入，系统在source字段正确记录每只股票的实际数据来源
- **双数据源完全失败**: 导入某只股票时，AkShare和Yahoo Finance都返回错误（如股票代码变更、已退市），系统跳过该股票，在错误日志中记录详细信息（两个数据源的错误消息），继续处理下一只股票，最终在导入报告中列出所有失败股票
- **交易日历差异**: 港股、美股和A股有不同的休市日（节假日不同），系统通过请求日期范围数据并根据数据源API实际返回的数据推断各市场的交易日，无需预先知道休市日
- **重复导入**: 用户误操作运行两次导入脚本，系统检测到股票代码已存在，使用upsert逻辑更新基本信息而非创建重复记录
- **部分导入失败**: 100只美股中有5只因为数据源问题导入失败，系统完成其他95只的导入，并生成失败报告，列出失败的股票代码和原因
- **导入中断恢复**: 导入进行到第150只股票时网络中断或系统重启，重新运行导入脚本后，系统从检查点恢复，跳过已成功导入的150只股票，继续处理剩余股票
- **港股美股混合搜索**: 用户在搜索框输入"Apple"，系统同时匹配美股"Apple Inc. (AAPL)"和可能的A股/港股中包含"Apple"的公司（如"苹果供应商"），按相关性排序显示结果
- **跨语言搜索**: 用户输入中文"苹果"搜索美股，系统能够匹配到"Apple Inc."；用户输入英文"Tencent"搜索港股，系统能够匹配到"腾讯控股"
- **货币单位显示**: 用户查看港股K线图时，价格轴显示"HKD"单位，美股显示"USD"单位，A股显示"CNY"或"¥"单位，避免混淆
- **VCP分析参数差异**: 港股和美股的波动性、流动性可能与A股不同，系统在执行VCP分析时使用统一的判断标准（收缩幅度、持续时间等），但需要考虑是否需要根据市场特性调整参数
- **核心股票定义**: 港股包含恒生指数成分股和恒生科技指数成分股（合计约110只，去重后），美股包含标普500成分股和纳斯达克100成分股（合计约550只，去重后）。系统优先导入这些市场的核心指数成分股。
- **数据历史长度**: 导入最近10年的历史K线数据（约2500个交易日），以支持长期趋势分析和VCP形态识别
- **数据源选择**: 使用多数据源策略 - 优先尝试AkShare获取港股和美股数据，如果AkShare不支持或数据质量不佳，则使用Yahoo Finance作为备选数据源，确保数据获取的稳定性和完整性

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须支持导入港股核心股票的基本信息（恒生指数和恒生科技指数成分股，约110只），包括股票代码、股票名称、上市日期、所属行业
- **FR-002**: 系统必须支持导入美股核心股票的基本信息（标普500和纳斯达克100成分股，约550只），包括股票代码、股票名称（保持英文原名）、上市日期、所属行业
- **FR-003**: 系统必须支持港股股票代码格式（如"00700.HK"、"09988.HK"），并能正确识别和存储
- **FR-004**: 系统必须支持美股股票代码格式（如"AAPL"、"GOOGL"、"TSLA"），并能正确识别和存储
- **FR-005**: 系统必须为港股股票导入最近10年的历史K线数据，包括日期、开盘价、最高价、最低价、收盘价、成交量、成交额
- **FR-006**: 系统必须为美股股票导入最近10年的历史K线数据，包括日期、开盘价、最高价、最低价、收盘价、成交量、成交额
- **FR-007**: 系统必须在Stock数据表的market字段中支持新的市场类型："HK"（港股）和"US"（美股），现有的"SH"和"SZ"保持不变
- **FR-008**: 系统必须记录每只港股和美股的货币单位（港股为HKD，美股为USD），以便在查询和展示时正确显示货币符号
- **FR-009**: 系统必须在导入过程中处理数据源错误（如API超时、限流、数据缺失），当两个数据源都失败时跳过该股票，记录详细错误信息（股票代码、尝试的数据源、错误原因、时间戳），并继续处理剩余股票
- **FR-010**: 系统必须在导入完成后生成详细统计报告，显示成功导入的股票数量、失败数量、总耗时，以及失败股票的详细清单（包括股票代码、尝试的数据源、失败原因）
- **FR-011**: 系统必须支持对港股和美股执行增量K线数据更新，只拉取最新缺失的交易日数据
- **FR-012**: 系统必须在股票列表和搜索结果中区分不同市场的股票，通过市场标识（"HK"、"US"、"SH"、"SZ"）帮助用户识别
- **FR-013**: 系统必须支持用户通过市场类型筛选股票列表（如只显示港股或只显示美股）
- **FR-014**: 系统必须支持对港股和美股执行VCP形态分析，使用与A股相同的分析算法和判断标准
- **FR-015**: 系统必须在查询和展示港股数据时，正确显示货币单位为"HKD"或"港币"
- **FR-016**: 系统必须在查询和展示美股数据时，正确显示货币单位为"USD"或"美元"
- **FR-017**: 系统必须为港股和美股提供与A股相同的功能支持，包括K线图查看、技术指标计算、收藏功能、绘图工具
- **FR-018**: 系统必须通过请求日期范围内的数据并根据数据源API实际返回的数据来推断各市场的交易日，无需预先维护交易日历
- **FR-019**: 系统必须为每只股票实现智能混合数据源策略，先尝试使用AkShare获取数据，如果失败则自动切换到Yahoo Finance获取该股票的数据
- **FR-020**: 系统必须在KLineData记录中标注实际使用的数据来源（source字段为"akshare"或"yahoo_finance"），以便追踪每只股票的数据来源和质量
- **FR-021**: 系统必须支持管理员手动触发指数成分股重新导入，当触发时系统重新获取最新的指数成分股列表，添加新成员并保留已移除股票的历史数据
- **FR-022**: 系统必须在导入过程中记录进度检查点，包括已成功导入的股票代码列表和当前处理状态
- **FR-023**: 系统必须支持从中断点恢复导入，重新运行导入脚本时自动跳过已成功导入的股票，继续处理剩余股票
- **FR-024**: 系统必须按数据源提供的原始语言存储和显示股票名称，港股使用中文名称，美股使用英文名称，不进行翻译转换
- **FR-025**: 系统必须支持用户使用中英文搜索股票，即使股票名称为英文，用户输入中文关键词（如"苹果"搜索"Apple"）时也能匹配到相关股票

### Key Entities

- **Stock (股票)**: 扩展现有股票实体，market字段新增"HK"（港股）和"US"（美股）两个枚举值，新增currency字段存储货币单位（"CNY"、"HKD"、"USD"）
- **KLineData (K线数据)**: 无需修改结构，通过stockCode关联港股和美股数据，source字段标注数据来源
- **ImportTask (导入任务)**: 记录港股和美股数据导入任务的执行状态，包括任务ID、市场类型、开始时间、结束时间、成功数量、失败数量、错误详情

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 系统能够在2小时内完成约110只港股的基本信息和10年历史K线数据的导入
- **SC-002**: 系统能够在4小时内完成约550只美股的基本信息和10年历史K线数据的导入
- **SC-003**: 用户能够通过股票代码或名称搜索港股和美股，搜索响应时间在1秒以内，准确率达到95%以上
- **SC-004**: 对港股和美股执行VCP分析的准确性与A股保持一致（95%以上准确率）
- **SC-005**: 用户在查看港股或美股K线图时，能够清晰识别货币单位（100%的查询结果正确显示HKD或USD）
- **SC-006**: 增量更新脚本能够在10分钟内完成所有港股和美股（合计200只以内）的最新K线数据更新
- **SC-007**: 导入过程中即使部分股票失败，系统仍能完成其余股票的导入，失败率低于5%
- **SC-008**: 用户能够在不参考文档的情况下，通过市场筛选功能快速定位港股或美股（用户任务完成率90%以上）

## Assumptions

- 核心股票范围：港股包含恒生指数和恒生科技指数成分股（约110只，去重后），美股包含标普500和纳斯达克100成分股（约550只，去重后）
- 历史数据范围：导入最近10年的历史K线数据（约2500个交易日），以支持长期趋势和形态识别
- 数据源策略：使用多数据源组合，优先尝试AkShare获取数据，如果AkShare不支持或数据质量不佳，则使用Yahoo Finance API作为备选方案
- 港股和美股的VCP形态分析使用与A股相同的算法和参数，不需要针对市场特性调整
- 系统性能足以支持三个市场的数据存储和查询（数据库容量约5-10GB，查询速度在可接受范围内）
- 数据更新频率为每日一次（在各市场收盘后），不需要实时或盘中数据
